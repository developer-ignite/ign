# Learning Analysis — Revision 2

## Mistake

CPT block placeholders ended with ellipsis ('...'), name fields lost spacing when empty, and the first name field was not focusable when its content was empty.

## Expected

Placeholders should be plain text without trailing punctuation. Space between first and last name fields should always be visible. All name fields should be focusable even when empty.

## Root Cause

Developer made reasonable but incorrect design interpretations:
1. Assumed placeholder text should follow form UI convention of ending with ellipsis ('...')
2. Used text-node spacer approach instead of flexbox gap for dual-field spacing, which became invisible when both fields were empty
3. Did not apply minimum width to RichText name fields, making them un-focusable when empty due to zero width

- **Type:** design-interpretation
- **Responsible Phase:** development

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Component Preferences**: CPT block inline editor placeholders must be plain text without trailing ellipsis or punctuation (e.g., 'First name' not 'First name...'). Placeholder text should be concise and action-oriented.
   - *Rationale:* Ellipsis in placeholders creates redundant visual noise in CPT block cards. Plain text improves authoring clarity and matches WordPress block editor conventions.

2. **[ADD] Component Preferences**: Dual inline RichText name fields must use flex gap-2 flex-wrap on the wrapper and inline-block min-w-[6rem] on each RichText component. This ensures the gap remains visible even when both fields are empty, and keeps both fields focusable for editing.
   - *Rationale:* Flexbox gap is always rendered, unlike text-node spacers which disappear when empty. Minimum width prevents zero-width fields that cannot receive focus.
