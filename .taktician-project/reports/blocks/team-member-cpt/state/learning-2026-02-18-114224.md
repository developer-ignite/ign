# Learning Analysis — Revision 1

## Mistake

Planning spec prescribed adding "photo" as a custom meta field (integer type), but the developer discovered the theme uses `inc/functions/featured-image.php` which registers `_thumbnail_id` for all post types including team_member, providing the same functionality without custom meta registration.

## Expected

The planning agent should recognize that featured image / post thumbnail for CPT blocks should use the theme's established focal point pattern (`_thumbnail_id` + `focal_point`) rather than a custom photo meta field.

## Root Cause

Missing project pattern documentation. The theme has a global featured image handling system in `inc/functions/featured-image.php` that registers `_thumbnail_id` and `focal_point` for all post types, but this pattern is not documented in taktician.md or the CPT block documentation. Planning agents assume custom meta fields unless explicitly told about the theme's existing pattern.

- **Type:** missing-project-pattern
- **Responsible Phase:** planning

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Required Patterns**: Featured Image for CPT Blocks — Always use the focal point post thumbnail pattern via inc/functions/featured-image.php instead of custom photo meta fields. For CPT block editors, use `ImageDropUploader` with `image={{ id: meta._thumbnail_id || featuredMediaId || null, focalPoint: meta.focal_point }}` and `onSelect={{ featured_media, meta: { ...meta, _thumbnail_id } }}` pattern (see resources/js/editor/featured-image.tsx and blocks/TeamMemberCPT/TeamMemberCPT.tsx for examples). Do NOT register custom photo/image meta fields — _thumbnail_id and focal_point are already registered globally for all post types.
   - *Rationale:* This pattern is already implemented in featured-image.tsx and the theme's featured-image.php registration. Documenting it prevents planning agents from creating specs with custom meta fields that duplicate existing infrastructure.

2. **[ADD] Component Preferences**: CPT Block Image Fields — For CPT block card previews with featured images, always use `ImageDropUploader` in the block body (inline editing) and `MediaUploadPanel` in the sidebar panel (canonical location). Both read/write to the same post meta via the focal point pattern. Never use custom photo meta fields — the theme's featured image system already handles this for all post types.
   - *Rationale:* Reinforces that ImageDropUploader is the component choice for CPT blocks (already documented in cpt-block.md section 5), and clarifies that this component works with the theme's global featured image system, not custom meta.
