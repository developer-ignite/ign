# Learning Analysis — Revision 1

## Mistake

Logo anchor rendered as empty link when logo ID was set but attachment was deleted or missing. No link wrapper should appear if no image output is produced.

## Expected

Anchor only renders when the logo image produces actual output. No empty anchors on the page.

## Root Cause

Initial implementation used only `! empty( $logoId )` to guard the anchor wrapper, but did not verify that `theme_output_svg_or_img()` actually produces output. When a media attachment is deleted or missing, the ID can still exist but produce no HTML, resulting in an empty `<a>` tag.

- **Type:** missing-project-pattern
- **Responsible Phase:** development

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Component Preferences**: When wrapping media output (images, SVGs, logos) in anchor tags or other containers, pre-check the actual HTML output before rendering the wrapper. Use the pattern: `$output = theme_function($id, false); if ( ! empty( $output ) ) { echo '<wrapper>' . $output . '</wrapper>'; }`. This prevents empty wrappers when referenced media is deleted.
   - *Rationale:* Media IDs can persist even when attachments are deleted; checking the actual output ensures only valid markup is rendered.
