# Learning Analysis — Revision 1

## Mistake

Address field displayed literal `<br>` text instead of rendering line breaks.

## Expected

RichText editor content with `<br>` tags should render as actual line breaks on the frontend.

## Root Cause

Inconsistent escaping function: `esc_html()` was used on the address field (line 59) while the same file already used `wp_kses_post()` for `hoursContent` (line 76) and `contactContent` (line 86). The developer didn't notice or follow the existing pattern in the same file.

- **Type:** missing-project-pattern
- **Responsible Phase:** development

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Required Patterns**: RichText editor content (with potential HTML tags like <br>, <strong>, etc.) must use wp_kses_post() for output, never esc_html(). Use esc_html() only for plain text fields (text inputs, metadata without HTML support).
   - *Rationale:* The Footer block already follows this pattern correctly for hoursContent and contactContent. Documenting this prevents developers from mixing escaping functions for similar content types in the same block.
