# Learning Analysis - Revision 1

## Mistake

Multiple workflow failures on the Archive block:
1. Skipped Design QA phase entirely after development
2. When QA was eventually run, it reported PASS without detecting that the pseudo-element vertical inset (32px from `-inset-8`) didn't match section padding (96px from `py-gap-8`)
3. Didn't update the block report after changes
4. Screenshots placed in wrong folder with wrong names
5. Didn't follow the full workflow documented in pipeline.md

## Expected

Full pipeline execution in order:
1. Development phase completes
2. QA phase runs with proper visual comparison (including pseudo-element coverage analysis)
3. Screenshots captured in correct folder (`.taktician-project/reports/blocks/archive/screenshots/`) with correct naming
4. Block report created or updated with changelog entry

## Root Cause

Two distinct failures:

**Orchestrator failure:** Skipped mandatory phases (QA, screenshots, report). Pipeline.md clearly states these phases apply to "All modes" but the orchestrator proceeded directly from development to presenting results.

**Design QA failure:** When eventually run, QA passed without detecting a structural mismatch. The dark-mode pattern documentation showed `md:before:-inset-8` (32px uniform inset), but the section had `py-gap-8` (96px vertical padding). The pseudo-element background didn't extend far enough vertically, leaving a visual gap. This relationship between inset values and section padding was not documented as a verification point.

- **Type:** ignored-instruction
- **Responsible Phase:** orchestrator, qa

   - *Replace:* The existing dark mode quick reference with uniform `-inset-8`
   - *With:* Updated pattern showing separate horizontal/vertical insets with explicit note that vertical inset MUST match section padding
   - *Rationale:* The original pattern used uniform `-inset-8` (32px) which doesn't account for larger section padding. Documenting that vertical insets must match prevents this mismatch.
