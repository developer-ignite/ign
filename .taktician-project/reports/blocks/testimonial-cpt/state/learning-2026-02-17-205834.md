# Learning Analysis — Revision 2

## Mistake

The orchestrator applied three code-related fixes directly to the codebase instead of delegating through the Developer Agent via the Task tool:
1. Replaced custom `VideoFileControl` component with `MediaUploadPanel` from `@taktdev/components` in both `TestimonialCPT.tsx` and `testimonial-cpt-panel.tsx`
2. Replaced inline tertiary arrow SVG markup with SVGR import: `import { ReactComponent as TertiaryArrow } from 'resources/images/tertiary-arrow.svg'`
3. Edited both files directly rather than spawning a Task subagent

## Expected

When user feedback identifies code issues to fix, the orchestrator should create a Task to delegate the fixes to the Developer Agent. The Developer Agent applies the changes, runs validation, and reports back. The orchestrator coordinates the pipeline but does not edit code files.

## Root Cause

The orchestrator skipped the Task delegation step and edited files directly, apparently viewing the fixes as "too simple" to warrant a subagent. This is the **third delegation violation in the same session** (revision 1 and 2 both show the pattern). The root cause is incomplete adherence to the pipeline's design constraint: **the orchestrator coordinates, agents execute**. Direct edits by the orchestrator bypass agent context, validation, and traceability.

- **Type:** ignored-instruction
- **Responsible Phase:** orchestrator

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Component Preferences**: Use `MediaUploadPanel` from `@taktdev/components` for file/video selection, never hand-build custom components with raw `MediaUpload` + `fetchMedia` + styled cards. MediaUploadPanel is the established project pattern and is used consistently across ContentWithMedia and other blocks.
   - *Rationale:* VideoFileControl was a custom hand-built component that duplicated MediaUploadPanel's functionality. Establishing MediaUploadPanel as the canonical pattern prevents future reimplementation of the same pattern.

2. **[ADD] Required Patterns**: All SVG icons and indicators must be imported via SVGR (`import { ReactComponent as IconName } from 'resources/images/icon.svg'`), never embedded as raw inline SVG markup. Webpack's `@svgr/webpack` loader and the `resources` alias are configured; use them. Exception: inline SVGs are only for programmatically generated graphics.
   - *Rationale:* The tertiary arrow SVG was inlined as raw markup instead of imported. SVGR is the project standard and was already configured; using it prevents duplication and makes icons easier to maintain.

### General (pending user approval)

ALL changes, including one-line edits, go through a Task subagent. This ensures traceability, agent context consistency, and prevents orchestrator context drift.
   - *Rationale:* Repeated delegation violations suggest agents view direct orchestrator edits as acceptable shortcuts. A clearer, more emphatic rule statement in the Phase 6 header would reinforce that delegation is mandatory, not optional.
