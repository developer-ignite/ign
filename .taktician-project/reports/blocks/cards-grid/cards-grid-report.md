## Block Behavior

**User-facing description of block functionality in layman's terms.**

### Layout & Structure

- **Grid Columns**: Displays cards in a configurable grid (1, 2, or 3 columns on desktop)
- **Responsive Adaptation**: Mobile shows 1 column (full width), tablet shows 2 columns, desktop shows the configured number
- **Section Header**: Optional header area with eyebrow text, main heading, and description using two-column layout
- **Card Structure**: Each card consists of an optional image area and a content area stacked vertically

### Content Fields

- **Section Header**:
  - Eyebrow: Short label text (hidden when empty)
  - Heading: Main section title (h2 element)
  - Description: Descriptive paragraph (hidden when empty)
- **Card Content** (per CardsGridItem):
  - Image: Optional image with 4:3 aspect ratio and focal point support
  - Title: Card heading (h3 element)
  - Description: Card description text
  - Button: Tertiary button with arrow icon (text and link URL)

### Visual Styling

- **Light Mode**: Clean white background with accent-colored image wrappers
- **Dark Mode**: Charcoal pseudo-element background with white text for high contrast
- **Card Images**: Rounded corners with padding, uses theme accent color as wrapper background
- **Equal Height**: Cards in each row stretch to match the tallest card

### Conditional Behaviors

- **Image Containers**: Hidden completely when no image is provided to a card
- **Button Area**: Only rendered when button has a label and URL
- **Eyebrow Text**: Hidden when empty; only shown when explicitly provided
- **Section Accessible Name**: Uses heading if provided, falls back to "Cards Grid" aria-label when heading is empty

## Development Notes

### Design Decisions

**Dark Mode Pattern Standardization:** The dark mode implementation follows the DynamicContentCarousel pattern:
- **Section**: Uses `dark bg-transparent!` classes to enable dark mode context
- **Container**: Uses `relative py-18 md:py-23` for vertical padding, combined with a charcoal pseudo-element
- **Pseudo-Element**: Styled with `before:absolute before:bg-charcoal before:rounded-3xl before:-z-1 before:-inset-x-(--side-gutter) before:inset-y-0 md:before:-inset-x-8`
- **Benefit**: Consistent dark mode spacing matching DynamicContentCarousel; increased vertical padding (72px/92px) provides better breathing room; pseudo-element is contained within container bounds (inset-y-0) rather than extending with negative inset

**PHP Attribute Extraction:** Removed redundant manual attribute extraction in favor of WordPress's auto-extracted variables:
- Auto-extracted variables like `$anchor`, `$darkMode`, `$eyebrow`, `$heading`, `$description`, `$columns` are available directly in the render function context
- Only derived/computed variables ($hasInnerBlocks, $hasHeader, $gridClasses, $hasImage, $hasButton) require manual assignment
- Uses `??` null coalescing operator for safe defaults: `$heading ?? null`

**Accessibility: Aria-Label Fallback:** The section element needs an accessible name at all times:
- When a heading is provided: Uses `aria-labelledby` auto-generated by `theme_block_props()` helper
- When heading is empty: Falls back to `aria-label='Cards Grid'` to ensure section always has an accessible name
- Follows the pattern established by CardsCarousel block

**Image Alt Text:** Images use WordPress's `wp_get_attachment_image()` function which respects media library alt text:
- Images are contextual decorations for cards (title and description provide content)
- Media library alt text is used when available
- Acceptable for accessibility since meaningful content is in title/description

- **Spacing Tolerance**: Minor 8px differences in grid gaps and internal spacing are within acceptable range and use standard theme tokens
