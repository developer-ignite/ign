# Learning Analysis — Revision 4

## Mistake

The header component lacked consistent top spacing between its default relative-positioned state and its fixed-positioned state. The JavaScript scroll behavior applied a 32px offset (`calculateFixedOffset()`) when the header became fixed, but the default state had no corresponding top padding. Additionally, there was no way to display the header as a pure overlay above content without affecting the layout flow of elements below.

## Expected

1. The header should have 32px top padding in its default state to maintain visual alignment at the page edge, consistent with the fixed-state behavior.
2. The header should support a `displayAboveContent` mode that uses absolute positioning with a negative bottom margin to overlay the content below without pushing it down in the document flow.

## Root Cause

The developer implemented spacing and offset calculations exclusively in the JavaScript scroll behavior handler (for fixed positioning) but did not sync that spacing to the CSS base state. The fixed-state 32px offset calculation existed in code but had no default CSS equivalent. Additionally, the overlay feature was a new requirement added during revision that required:
- A new boolean attribute (`displayAboveContent`)
- A new layout pattern combining absolute positioning with negative margin
- Conditional JS behavior (spacer logic should skip when overlay mode is active)

This is a **design-interpretation** mistake: the initial spec didn't specify how multiple positioning modes should maintain visual consistency, and the overlay pattern wasn't part of the original plan.

- **Type:** design-interpretation
- **Responsible Phase:** development

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Layout Patterns**: When implementing header components with multiple positioning modes (relative, fixed, absolute overlay), ensure consistent spacing across all modes. If fixed positioning includes a calculated offset (e.g., 32px), apply that same spacing to the default relative state via top padding/margin. Overlay modes use negative margin matching the element's height to collapse document flow impact.
   - *Rationale:* Headers with fixed and relative states must maintain visual alignment at the layout edge to prevent spacing inconsistencies when toggling between modes.

2. **[ADD] Component Preferences**: Boolean attributes controlling visual positioning (fixed, sticky, overlay) should include a `display*` prefix for clarity. Example: `displayAboveContent` for overlay mode, `displayFixedOnScroll` for fixed headers. This distinguishes layout-controlling attributes from feature toggles.
   - *Rationale:* Explicit naming prevents confusion between behavioral toggles and visibility controls, and improves discoverability in inspector panels.
