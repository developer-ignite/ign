# Learning Analysis — Revision 3

## Mistake

1. Logo size classes used `**: ` (all descendants) selector instead of `*:` (direct children), unintentionally targeting SVG internals (paths, groups, etc.).
2. Fixed state activated when the header element itself reached the viewport top, instead of when the margin area above the element reached the viewport top.

## Expected

1. Logo sizing classes should use `*:` direct child selector to target only the SVG wrapper, not internal SVG structure.
2. Fixed mode should activate when the header's margin area reaches the viewport top, requiring the fixed-state threshold to account for margin-top spacing.

## Root Cause

1. **SVG Selector Anti-Pattern** — The Developer Agent applied the standard descendant-selector pattern (`**:`) used for styling nested HTML elements. However, SVG elements have internal structure (paths, groups, fills) that shouldn't be affected by sizing constraints meant for the SVG wrapper. The agent didn't anticipate SVG-specific behavior because it wasn't explicitly documented.

2. **Incomplete Fixed-State Logic** — The agent calculated the fixed-positioning threshold based on when the element reaches the viewport top. However, headers commonly have margin-top that creates a margin area above the element. The user expected fixed mode to activate when this margin area reaches the top, requiring subtraction of marginTop from the scroll calculation. This Header-specific behavior wasn't explicit in the development spec.

- **Type:** design-interpretation
- **Responsible Phase:** development

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Component Preferences**: Logo sizing in headers
   - *Content:* Use direct child selector `*:` (not `**:`) for sizing classes on SVG wrappers to avoid targeting internal SVG elements (paths, groups, etc.). Descendant selectors will apply size constraints to nested SVG internals, breaking icon rendering.
   - *Rationale:* Prevents accidental styling of SVG internal structure when applying sizing to logo containers.

2. **[ADD] Component Preferences**: Fixed positioning calculation in Header block
   - *Content:* When implementing sticky/fixed header behavior, account for the header's margin-top in the fixed-state threshold calculation. Subtract marginTop from the element's scrollPosition to determine when the margin area (not the element itself) reaches the viewport top. Fixed mode should activate earlier to preserve margin spacing.
   - *Rationale:* Ensures fixed positioning activates at the correct scroll position, accounting for header margin as part of the active region rather than trigger point.
