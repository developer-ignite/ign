# Learning Analysis — Revision 1

## Mistake

The logo was not visible on the rendered frontend (black text on dark background) and the logo position was stretched to full header height instead of being vertically centered. However, Design QA marked the block as PASS.

## Expected

The logo should be visible with proper white color on the dark background, and positioned vertically centered (not stretched) as shown in the Figma design.

## Root Cause

Two separate issues converged to cause this failure:

1. **Inline SVG currentColor inheritance:** The logo uses inline SVG rendering via `theme_output_svg_or_img()`. Inline SVGs use `currentColor` which inherits from the CSS `color` property, not from visual context. The logo container lacked an explicit `text-white` class, causing SVG paths to inherit black text color and become invisible on the dark background.

2. **Logo layout:** The logo container used `self-stretch` instead of `self-center`, making it fill the entire header height instead of remaining proportional.

3. **Verification failures:** Neither the Planning Agent nor Design QA caught these issues. The Planning Agent said "no code changes needed" without understanding inline SVG color requirements. Design QA reported PASS without rigorously verifying that the logo was actually visible in screenshots (this should have been obvious during visual comparison).

- **Type:** missing-project-pattern
- **Responsible Phase:** design (QA failure) + planning (validation failure)

## Suggestions

### Project-Specific (applied to taktician.md)

1. **[ADD] Required Patterns**: Inline SVG Color Rendering
   - *Rationale:* This prevents silent rendering failures where SVG logos appear invisible on dark backgrounds due to currentColor inheriting black text color. The pattern is project-specific because it relates to the theme's use of `theme_output_svg_or_img()` for inline SVG rendering.

### General (pending user approval)

1. **[ADD] .taktician/agents/design-agent.md > Step 3: Detailed Visual Comparison**: Visual verification must confirm element visibility
   - *Rationale:* Design QA marked invisible elements as passing because it checked structure without verifying visibility. This rule ensures QA performs visual verification of actual rendered output, not just code structure.

2. **[ADD] .taktician/agents/planning-agent.md > Phase 2: Analyze Figma Design**: Inline SVG color pattern check
   - *Rationale:* This prevents Planning Agent from validating specifications that depend on inherited text color through the DOM tree, which fails when currentColor rendering is involved.

---

## Summary

The revision cycle revealed a compound failure:

1. **Planning Agent gap:** Did not recognize or verify that inline SVGs require explicit color class on their container
2. **Design QA gap:** Did not perform rigorous visual verification to confirm the logo was actually rendered/visible in screenshots
3. **Project knowledge gap:** The inline SVG currentColor pattern was not documented in project rules

One project-specific rule has been applied to prevent recurrence. Two general rules are pending user approval to improve Planning Agent and Design QA workflows.
